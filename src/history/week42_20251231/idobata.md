## 未完了タスクで進行中の議論

### [PR #429](https://github.com/digitaldemocracy2030/idobata/pull/429)：MCPサーバをバイパスして直接Octokit（GitHub API）を呼び出す実装

- **作成者**: kuboon+Devin  
- **更新日**: 2025-07-18（+889行 / -29行、計8ファイルを変更）
- **主な変更内容**:
  - **直接GitHub連携**: MCPサーバを経由せずにGitHubのAPIを呼び出すためのOctokit依存関係と認証処理を追加  
  - **ロジック移行**: GitHub操作処理を`policy-edit/mcp/src/`から`policy-edit/backend/src/`に移動  
  - **サービスのリファクタリング**: `IdobataMcpService`を改修し、GitHubツールを直接呼び出す形に変更  
  - **MCPのnpmワークスペース除外**: CIが通らない問題を解決するため、MCPをnpmワークスペースから除外  
  - **環境変数の追加**: GitHub App用の認証情報を設定するために新しい環境変数を導入  

- **議論のポイント**:
  - **環境変数の設定**: `GITHUB_APP_ID`や`GITHUB_INSTALLATION_ID`といったGitHub App向けの環境変数を正しく設定しないと動作しないため、デプロイ環境別に丁寧な検証が必要  
  - **プライベートキーの管理**: `/app/secrets/github-key.pem`のパーミッション設定や秘匿管理をどう行うか  
  - **Error Handling**: GitHub API呼び出し時にエラー発生した際の処理フロー検証  
  - **パフォーマンス向上**: MCPを経由しないことで応答速度が上がる見込みだが、引き続きテストとフィードバックが求められる  

- **今後の参加方法**:
  - 実際にこのPRブランチをチェックアウトしてテストフローを試し、問題や改善点があればぜひコメントや提案をください。  
  - デプロイ面やセキュリティ面のレビューなど、幅広い視点でのご意見を歓迎しています。
