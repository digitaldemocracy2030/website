## 未完了のタスクと議論の状況

### 1. [PR #727](https://github.com/digitaldemocracy2030/kouchou-ai/pull/727) - 静的ビルド実行前に公開状態のレポートが存在するかを検証

- 作成者: NISHIO+Devin  
- 概要:  
  - 静的エクスポートビルドを実行する際、公開状態のレポートが一つもないと分かりにくいエラーが出てしまう問題を解決する取り組みです。  
  - 公開レポートが存在しない場合に、わかりやすい日本語のエラーメッセージを表示してビルドが止まるように修正されています。

- 関連するIssue: [Issue #726](https://github.com/digitaldemocracy2030/kouchou-ai/issues/726)
- 変更内容のポイント:  
  - 事前スクリプト（`validate-reports.mjs`）で、APIからレポート一覧を取得しチェックする仕組みを追加。  
  - 公開レポートがない場合は日本語で詳細な対処方法を提示し、エラー時にビルドを中断。  
  - 実運用ではAPIサーバーが停止している可能性もあるため、その場合も別のメッセージを表示する工夫があります。

- 議論のポイント:  
  - 実際の環境で公開レポートが存在しない状態での最終テストが未実施とのこと。  
  - APIの不具合によりテストが制限されている部分があり、引き続き動作確認とレビューが必要だとしています。

### 2. [PR #722](https://github.com/digitaldemocracy2030/kouchou-ai/pull/722) - Add filesystem-based usage documentation and validation tests

- 作成者: NISHIO+Devin  
- 概要:  
  - APIサーバーを起動せずに、ファイルシステムベースでパイプラインを実行できる機能に関して、詳細なドキュメントとスキーマ検証のテストが追加されました。  
  - 設定ファイル、入力CSV、出力JSONなどをPydanticでスキーマ定義し、CLIでのバリデーションが可能になっています。

- 関連するIssue: [Issue #721](https://github.com/digitaldemocracy2030/kouchou-ai/issues/721)
- 変更内容のポイント:  
  - ドキュメント (FILESYSTEM_USAGE.md) を388行にもわたって大幅に追加し、利用手順や検証方法を丁寧に説明。  
  - CLIオプション（`--validate-input`, `--validate-config`, `--validate-output`, `--dry-run`）が追加され、データの妥当性チェックが容易に。  
  - 54のテストケースが追加され、細かな動作検証が可能に。

- 議論のポイント:  
  - テストはスキーマ検証を中心にしているため、実際のAPIキーを用いたパイプライン動作は未確認。  
  - リスクの高い箇所（`hierarchical_main.py`の早期returnなど）のレビューが求められています。  
  - CLIの挙動が既存フローを壊していないか、人手による確認も推奨されています。
